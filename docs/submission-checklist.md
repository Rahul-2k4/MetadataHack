# Submission Checklist

## Must-Have

- [ ] README clearly states problem, approach, architecture, and setup.
- [ ] Demo script is rehearsed and fits time budget.
- [ ] OpenMetadata usage is explicit and visible in the demo.
- [ ] Repo includes only hackathon-relevant code/docs/assets.

## Quality Gates (Judging Alignment)

- [ ] Impact claim is concrete and not generic.
- [ ] Innovation is clearly differentiated from baseline implementations.
- [ ] Technical choices are documented and defensible.
- [ ] UX walkthrough is coherent and easy to follow.
- [ ] Presentation quality (README + demo flow) is polished.

## Final Pre-Submit Checks

- [ ] All links work.
- [ ] Setup steps run on a clean machine.
- [ ] Screenshots/diagrams match current build.
- [ ] Submission artifacts are in final, reviewable form.
