# Hackathon Rules (Repo Snapshot)

Snapshot date: 2026-04-15
Primary source: https://www.wemakedevs.org/hackathons/openmetadata/rules

## Core Rules

1. Team size is 1-4 members.
2. Project must integrate with or extend OpenMetadata.
3. For contribution-track issues, get assignment before starting work.
4. Starter templates/codebases are allowed, but minimal-change resubmissions are not.
5. Third-party tools/libraries/APIs/assets are allowed.
6. Planning is allowed before kickoff; coding/design should begin after hackathon start.
7. Project must be software-first (hardware optional).
8. AI tools are allowed only with explicit disclosure in submission.
9. Follow Code of Conduct; violations can disqualify.
10. Rule violations may result in disqualification.

## Additional Policy Notes

- IP created during the hackathon belongs to the creating team.
- Career/interview opportunities at Collate are possible but not guaranteed.

## Team Working Interpretation

- Treat this file as the repo-level interpretation of official rules.
